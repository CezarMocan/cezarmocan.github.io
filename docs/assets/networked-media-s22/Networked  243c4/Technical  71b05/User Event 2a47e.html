User Events & Interaction

We are finally here ‚Äî interaction, the last missing part in our frontend web development toolkit. Once you‚Äôve covered this material, you will technically know everything you need about HTML, CSS and Javascript in order to build relatively complex (and functional) static websites. (static doesn‚Äôt mean no interaction, or fun, it just means no backend or dynamic data. The HTML you wrote is all the website has.)

As mentioned in the Web Time tutorial, Javascript is an event-based programming language, meaning that it allows us to easily implement certain logic (read execute a function) when certain events happen.

When talking about time on the web, those events had to do with time. setTimeout's event can be verbally described as X seconds in the future. setInterval's event can be verbally described as starting now, every X seconds.

In this tutorial we will look at user-generated events, things like when the user clicks on this <div>, or when the user types the letter "X" on their keyboard.

jQuery‚Äôs on method

The on method attaches one or more event handlers for the selected elements and child elements. Event handlers attached using the on method will work for both current and FUTURE elements (like a new element created by a script). As of jQuery 1.7, all events are bound via the¬†on¬†method, whether you call it directly or whether you use an alias/shortcut method such as¬†bind¬†or¬†click, which are mapped to the¬†on¬†method internally. (more)

The general syntax for attaching an event listener to an object using the on method is like this:

$(selector).on(eventName, callbackFunction)

  • The selector works as discussed in a previous tutorial (by tag name, by id, by class, etc.)
  • The eventName needs to take specific values, which are defined by HTML & Javascript. Examples include click, keypress, hover, and more. You can find a full list of DOM events here.
  • The callbackFunction is a function that we define, similarly to timers. This is where we define what we want to happen as a result of the interaction (click, key press, etc.) and gets called when Javascript detects that user interaction on the given selected elements.

Mouse events

The click event

$("#element").on("click", function(event) { /* code here */ } )

First of all, let‚Äôs talk about events. ‚Äúclick‚Äù is an event that is bound via the¬†on¬†method. All the different visitors' actions that a web page can respond to are called events, and jQuery responds to events in an HTML page. Keep in mind that an event represents the precise moment when something happens. There could be one or more space-separated event types and optional namespaces, all events should be contained in a set of quotation marks.

Meanwhile, function( ) is a handler, it executes when an event (in our case, the ‚Äúclick‚Äù event) is triggered and completed. We need it because JavaScript statements are executed line by line. However, with effects, the next line of code can be run even though the effect is not finished. This can create errors. To prevent this, you can create a callback function, which is executed after the current effect is finished.

For instance, the example below has no callback parameter, and the alert box will be displayed before the hide effect is completed:

$("button").on("click", function(event) {
  $("p").hide(1000); // without function, "alert" happens before "hide"
  alert("The paragraph is now hidden");
});

The example below has a callback parameter that is a function (‚Äùalert‚Äù) which will be executed after the hide effect is completed:

$("button").on("click", function() {
  $("p").hide("slow", function(event) { // with funciton, "alert" doesn't happen until "hide" is completed 
    alert("The paragraph is now hidden");
  });
});

  • The event that is contained in function is an event object. When an event is triggered, jQuery passes the handler an event object that it can use to analyze and change the status of the event. This object includes important properties and methods for cross-browser consistency, e.g. target, pageX, pageY, relatedTarget etc.

    For example, event.type contains the event name:

    $(document).ready(function(){
  $("button").on("click", function(event) {
    alert(event.type); // it will return an alert box with the event type "click"
  });
});

And here is a full example of how you would use this on a page:

      alert("You just clicked me!");

The mouseenter event

$("#element").on("mouseenter", function(event) { /* code here */ } )

The function is executed when the mouse pointer enters the HTML element:

$("p").mouseenter(function(event){
	// text's background becomes yellow when the cursor enter the p element
  $("p").css("background-color", "yellow");
});

There is also a mouseover event. Note the difference between the two:

  • The mouseenter event is triggered only when the mouse pointer hits the element. The counterpart event is¬†mouseleave. It does not propagate up the document hierarchy.
  • The mouseover event triggers when the mouse pointer enters an element and any one of its child elements. Its counterpart is¬†mouseout.

The mouseleave event

$("#element").on("mouseleave", function(event) { /* code here */ } )

The mouseleave event occurs when the mouse leaves the selected element.

$(document).ready(function(){
  $("p").on("mouseenter", function(event){
    $("p").css("background-color", "yellow"); 
		// text's background becomes yellow when the cursor enter the p element
  });
  $("p").on("mouseleave", function(event){
    $("p").css("background-color", "lightgray"); 
		// text's background becomes grey when the cursor leaves the p element
  });
});

Note:¬†Unlike the¬†mouseout¬†event, the mouseleave event only triggers when the mouse leaves the selected elements. The mouseout event is triggered if a mouse leaves any child elements as well as the selected element.

The mousemove event

$("#element").on("mousemove", function(event) { /* code here */ } )

The mousemove event is triggered each time the mouse pointer is moved when it is over an element. Note that the it occurs each time when you move the mouse by one pixel.

$(document).ready(function(){
  $(document).mousemove(function(event){
    $("span").text(event.pageX + ", " + event.pageY); // shows the position of mouse pointer on time
  });
});

Keyboard events

The keydown event

$(document).on("keydown", function(event) { /* code here */ } )

The keydown event occurs when a keyboard key is pressed down (as opposed to when it‚Äôs released.)

If you attach this event listener to the whole document, it‚Äôll listen to key presses anywhere on the page. If you attach it to an individual element (like an <input> field,) it‚Äôll only listen to key presses when that element is focused.

$(document).ready(function(){
	$("input").on("keydown", function(event){
			// Change the background color of an <input> field to yellow when a keyboard key is pressed down
	    $("input").css("background-color", "yellow"); 			
			// Print to the console what key was pressed
			console.log(event.key)
	})
})

You can use the event.key¬†property to look at which keyboard key was pressed. There are actually three properties on the event object relating to the key that was pressed:

  • event.key gives you the character that was typed (a, b, c, d, etc.)
  • event.which gives you the ASCII code of that character; for instance, if you type ‚Äúj‚Äù, it will return ‚Äú74‚Äù.
  • event.code gives you the HTML code of that character; for instance, if you type ‚ÄúJ‚Äù, it will return ‚Äú&#74;‚Äù

The keyup event

$(document).on("keyup", function(event) { /* code here */ } )

The keyup event occurs when a keyboard key is released. 

$(document).ready(function(){
	$("input").on("keydown", function(event) {
		// Change the background color of an <input> field to yellow when a keyboard key is pressed down
    $("input").css("background-color", "yellow"); 
  });

	$("input").on("keyup", function(event) {
		// Change the background color of an <input> field to pink when a keyboard key is released
    $("input").css("background-color", "pink"); 
  });
});

Note the order of events related to the key event:

  1. keydown - The key is on its way down
  1. keypress¬†- The key is pressed down
  1. keyup¬†- The key is released
