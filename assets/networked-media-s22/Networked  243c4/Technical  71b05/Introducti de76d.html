Introduction to the DOM & JQuery

The main use of Javascript in the web browser is its ability to create dynamic and interactive webpages. So far, we‚Äôve been looking at HTML and CSS, and you‚Äôve probably noticed that the content is static: the browser will display all the elements that were defined within the HTML file, with the CSS styles applied to them, and nothing else. There is no way for a website‚Äôs content to change dynamically (e.g. in response to a user action) by simply using HTML and CSS. We need Javascript for that.

The Document Object Model (DOM)

When running in the browser, Javascript has access to the Document Object Model, or the DOM of an HTML page. This is a representation of the entire HTML structure of the page, which Javascript can manipulate in order to make structure, content or styling changes. It has a tree-like structure (remember, HTML is made up of nested tags,) something like the image below:

With the object model, JavaScript gets all the power it needs to create dynamic HTML:

  • JavaScript can change all the HTML elements in the page
  • JavaScript can change all the HTML attributes in the page
  • JavaScript can change all the CSS styles in the page
  • JavaScript can remove existing HTML elements and attributes
  • JavaScript can add new HTML elements and attributes
  • JavaScript can react to all existing HTML events in the page (user events, such as clicking or key presses, we will talk about them in a future tutorial)
  • JavaScript can create new HTML events in the page

This means that anything you could create, change or remove manually when writing HTML or CSS can also be manipulated in Javascript, through the DOM.

Adding Javascript to a website

Before working with the DOM, we need to first be able to import a Javascript file into our websites. This is pretty similar to how we import stylesheets, but instead of using the link tag we use the script tag.

<html>
	<head>
		<script src="/script.js"></script>
	</head>
	
	<body>
		Nothing to see here.
	</body>
</html>

And, we can write a simple piece of Javascript code which just alerts us of the fact that it was imported in the page.

alert('Hello this is Javascript speaking!')
console.log('This is a message in the console')

In order for the two files above to work, make sure both of them live in the public directory within the web servers on your Digital Ocean droplet.

The script above does two things: first, it opens a pop-up in your browser, saying Hello this is Javascript speaking. Second, it prints a message in the developer console (which you can open by right clicking anywhere on your page, selecting the Inspect Element option, and navigating to the Console tab in the newly opened sub-window.)

jQuery

Manipulating the DOM can be done using pure Javascript (through using the document and window global variables.) However, for this class we will use an external library called JQuery in order to make our lives easier. JQuery offers a more succinct syntax for interacting with the DOM, as well as a handful of really fun utility functions for doing that.

Since JQuery is an external library, we need to tell our HTML page where to import its code from. We do that within <head>, with the same <script> tag that we use to import our own script.

<html>
	<head>
		<script src="https://code.jquery.com/jquery-3.6.0.min.js"></script> <!-- This is the jquery script -->
		<script src="/script.js"></script> <!-- This is our own script -->
	</head>
</html>

In case you are wondering where I got the link within the src attribute from, it‚Äôs on the JQuery website (jQuery Core ‚Üí jQuery 3.x ‚Üí jQuery 3.6.0 ‚Üí minified). Whenever you want to use jQuery in your code, you need to include this script tag, with that same URL for the source attribute.

Waiting for the DOM to load

Interacting with the DOM can only happen once the HTML content of the page has loaded. Since our JS script might load before the actual HTML of the page, we need a way of listening for when the HTML page has completed loading everything. Fortunately, there is an easy solution for that:

$(document).ready(function() {
	console.log('The HTML has finished loading, so we can start interacting with the DOM.')
})

Most of your frontend Javascript code will live in the function above, since that guarantees the DOM is fully loaded.

In terms of syntax, we have a few things going on:

  • $ ‚Üí this is simply how we access the jQuery library. This is funky syntax, most Javascript libraries use full words for their names, but the jQuery developers chose the dollar sign in order to make writing code shorter. It might seem a bit confusing right now, but we will collectively thank them later.
  • $(selector) ‚Üí This is how we tell jQuery to access a specific element or set of elements in the DOM. The selector can be a tag ($("p"),) an id ($("#my-id"),) a class ($(".my-class")) or the entire document, which is what we‚Äôre doing above. Selecting by tag, id or class works similarly to how CSS operates: all HTML elements that match the given selector will be returned, and all further code will be applied to all of them. Using the document selector is a somewhat special usecase (notice in the code above it‚Äôs not surrounded by quotes,) and it‚Äôs what allows us to detect whether the page has already loaded or not.
  • $(selector).operation() ‚Üí once we‚Äôve chosen our selector, we apply an operation to the elements that were returned by this selector. You will see a few examples of this in a further section of this document. In the example above, the operation is called ready, which is actually an event listener. So, describing the code above verbally would look something like this: ‚ÄúHey jQuery, please listen to what document has to say, and whenever she sends the signal for ‚Äúready‚Äù, run this custom function which prints something to the console‚Äù. The ready event listener is what tells us that the DOM has loaded.

Different types of selectors; changing the content of an element (html())

Let‚Äôs look at three different types of selectors. We‚Äôll start with the following HTML page:

<html>
	<head>
		<script src="https://code.jquery.com/jquery-3.6.0.min.js"></script> <!-- This is the jquery script -->
		<script src="/script.js"></script> <!-- This is our own script -->
	</head>

	<body>
		<p> This is a paragraph </p>
		<p id="importantParagraph"> This is an important paragraph </p>
		<p class="blue-paragraph"> This is a blue paragraph </p>
		<p class="blue-paragraph"> This is another blue paragraph. </p>
	</body>
</html>

We have four paragraphs. One of them has no attributes, one of them has an id, and the other two have an identical class attribute. The code below uses the html() function inside jQuery in order to change the content of our paragraphs. Using different selectors, we can achieve different outcomes:

$(document).ready(function() {
	$("p").html("All paragraphs now have this text") // Selecting by tag is done by simply using the name of the tag, surrounded by quotes.
	$("#importantParagraph").html("Only the second paragraph has this text") // Selecting by ID is done by using the "#" symbol, followed by the ID of the element we're working with.
	$(".blue-paragraph").html("The last two paragraphs have this text") // Selecting by class is done by using the "." symbol, followed by the class name of the element(s) we're working with.
})

Changing the styling of an element (css() and addClass())

We can use jQuery to directly specify CSS properties for the elements we have selected. In the example below, we make the paragraphs which have the blue-paragraph class actually have blue text (on a pink background.)

<html>
	<head>
		<script src="https://code.jquery.com/jquery-3.6.0.min.js"></script> <!-- This is the jquery script -->
		<script src="/script.js"></script> <!-- This is our own script -->
	</head>

	<body>
		<p> This is a paragraph </p>
		<p id="importantParagraph"> This is an important paragraph </p>
		<p class="blue-paragraph"> This is a blue paragraph </p>
		<p class="blue-paragraph"> This is another blue paragraph. </p>
	</body>
</html>
$(document).ready(function() {
	$(".blue-paragraph").css("color", "blue") // The css() function takes two parameters: the CSS property we want to change, and the value we want to give it.
	$(".blue-paragraph").css("background-color", "pink") // You can call the css() function multiple times for changing different properties.
})

We can also use jQuery to assign a class to an element that doesn‚Äôt already have it, or conversely, to remove a class from an element which does have it. For the former, we could write the code below in order to programatically give the importantParagraph a blue-paragraph class as well. The addClass function 

$(document).ready(function() {
	$("#importantParagraph").addClass("blue-paragraph") // We select by ID, and add the blue-paragraph class to the paragraph which has the given id.
	$(".blue-paragraph").css("color", "blue") // Same code as above, we change the styling of the paragraphs which have this class.
})

Creating and removing HTML elements (appendTo, remove)

We can also use jQuery in order to create entirely new elements, or remove existing elements.

I‚Äôll start with removing, since it‚Äôs easier. Once we figure out our selector, we simply call .remove() on the value returned by the selector in order to entirely remove the element from the DOM. For removing all paragraphs with the blue-paragraph class in the example above, we would simply call

$(".blue-paragraph").remove()

Creating a new element is a two-step process. First, we need to know what type of element we want, and what its corresponding HTML tag is. Second, we need to find an already existing element on the HTML page, and add our newly created element to it. Remember, HTML has a tree-like structure, and if we don‚Äôt add our new element to this tree, as another element‚Äôs child, the newly created element will simply not show up.

<html>
	<head>
		<script src="https://code.jquery.com/jquery-3.6.0.min.js"></script> <!-- This is the jquery script -->
		<script src="/script.js"></script> <!-- This is our own script -->
	</head>
	
	<body>
		<div id="container">

		</div>
	</body>
</html>

Let‚Äôs use this HTML page, and use jQuery to add a paragraph inside of the container div.

$(document).ready(function() {
	var newElement = $("<p>This is a blue paragraph.</p>").appendTo("#container")
	newElement.css("color", "blue")
})

Notice a few things:

  • We are using the $() function differently than we have been so far. Instead of passing it a selector, we simply write the HTML declaration of the element we want to add. Internally, jQuery will recognize whether we are trying to use a selector, or trying to create a new element, based on the syntax (a new element will always be an html tag)
  • We are calling the appendTo function, and the argument of that function is the selector for the id container. That‚Äôs the div we want to add our paragraph to.
  • We are storing the newly created element in a variable, and in the second instruction we are updating the CSS of that element.
    • We could have also chained all three commands, like this:
    var newElement = $("<p>This is a blue paragraph.</p>").appendTo("#container").css("color", "blue")
